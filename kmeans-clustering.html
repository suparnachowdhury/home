Understanding K-Means Clustering:
A Practical Guide

A comprehensive walkthrough — from algorithm intuition to feature engineering and final cluster profiling using real-world city lifestyle data.

In machine learning, K-Means clustering is one of the most popular algorithms for uncovering hidden patterns in data. It is an unsupervised learning technique — meaning it does not rely on labeled data — and is widely used for customer segmentation, image compression, and pattern recognition.

01 What Is K-Means Clustering?

K-Means clustering splits a dataset into K distinct groups (clusters). The goal is to group data points that are similar to each other while keeping different clusters as distinct as possible. The term "means" refers to the average position of points in each cluster — known as the centroid.

Core Idea

Every data point belongs to the cluster whose centroid is closest to it. The algorithm iteratively refines these assignments until the cluster boundaries stabilize.

02 How It Works

  1. Choose K — decide how many clusters you want to form.
  2. Initialize centroids — randomly select K data points as starting centers.
  3. Assign points — each data point is assigned to the nearest centroid (typically by Euclidean distance).
  4. Update centroids — recalculate each centroid as the mean of all points assigned to it.
  5. Repeat — continue reassigning and updating until clusters stabilize (no major changes).

03 Implementing K-Means in Python

We use Python's scikit-learn library. Let's start with the model initialization:

Python 
from sklearn.cluster import KMeans

# Initialize KMeans with 2 clusters
kmeans = KMeans(n_clusters=2, n_init='auto', random_state=42)

Key parameters explained:

  • n_clusters=2 — the number of clusters K to form. Default in scikit-learn is 8.
  • n_init='auto' — controls how many times the algorithm restarts with different centroid seeds to avoid local minima. With k-means++ (default init), it runs once; with 'random', it runs 10 times.
  • random_state=42 — seeds the random number generator so results are reproducible.

04 Why Feature Scaling Matters

K-Means uses Euclidean distance to assign points to clusters. Features with larger ranges will dominate the calculation, skewing results.

FeatureRangeStd DevImpact Without Scaling
population_density100 – 14,427~2,983Dominates distance
happiness_score2.5 – 8.5~1.69Effectively ignored

By applying StandardScaler, all features are brought to mean 0 and std 1, ensuring equal contribution to the distance metric.

Python · Feature Scaling 
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_scaled = scaler.fit_transform(data)

06 Finding the Optimal K — Elbow Method

Inertia measures how compact clusters are — the sum of squared distances between each point and its centroid. K-Means minimizes this. Since inertia always decreases as K grows, we use the Elbow Method to find the inflection point where additional clusters yield diminishing returns.

Key Observation

With K=2, inertia = 1411.05. Increasing to K=3 drops it to 1036 — a meaningful gain. Beyond K=5, improvements become marginal. Two candidate elbows appear at K=3 and K=5.

Python · Elbow Method 
inertia_values = []
k_values = range(2, 12)

for k in k_values:
    kmeans = KMeans(n_clusters=k, random_state=42)
    kmeans.fit(X_scaled)
    inertia_values.append(kmeans.inertia_)

plt.plot(k_values, inertia_values, marker='o')
plt.xlabel('Number of Clusters (K)')
plt.ylabel('Inertia')
plt.title('Elbow Method for Optimal K')
plt.show()

07 Feature Engineering for Better Clusters

Raw features may not capture the true structure of the data. We create composite features that encode domain knowledge about livability and economic efficiency:

Python · Feature Engineering 
# Composite features encoding domain knowledge
data['income_per_density']   = data['avg_income'] / data['population_density']
data['rent_to_income_ratio']  = data['avg_rent'] / data['avg_income']
data['urbanization_score']    = (
    data['population_density']    * 0.3 +
    data['public_transport_score'] * 0.3 +
    data['internet_penetration']   * 0.2 -
    data['green_space_ratio']      * 0.2)
data['quality_of_life']       = (
    data['happiness_score'] + data['green_space_ratio'] +
    data['public_transport_score']) / 3
data['environment_comfort']   = (
    (100 - data['air_quality_index']) + data['green_space_ratio']) / 2
data['mobility_index']        = (
    data['public_transport_score'] / data['population_density'])

# Handle infinities, scale
data.replace([np.inf, -np.inf], np.nan, inplace=True)
data.fillna(data.mean(), inplace=True)

features_to_scale = ['income_per_density', 'rent_to_income_ratio',
                     'urbanization_score', 'quality_of_life',
                     'environment_comfort', 'mobility_index']

scaler    = StandardScaler()
X_scaled_v2 = scaler.fit_transform(data[features_to_scale])

09 Key Takeaways

  • Feature scaling is non-negotiable — without it, high-variance features like population_density completely dominate the distance calculation.
  • K selection requires judgment — the Elbow Method guides but does not dictate. Domain knowledge and interpretability matter.
  • Feature engineering unlocks richer structure — composite features encoding ratios and weighted scores produced cleaner, more meaningful clusters than raw features alone.
  • PCA helps visualize but not model — use it for plotting cluster structure, not as input to K-Means (unless dimensionality is genuinely very high).
